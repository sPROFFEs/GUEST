Internal hosts

Devices on the LAN behind this gateway. Discovered via DHCP leases + ARP.

How to read the table
  • Reserved = OFF (dynamic): the host gets whatever IP the DHCP pool hands it. The IP shown here mirrors the current lease and updates on every scan — you can't edit it, since dnsmasq decides.
  • Reserved = ON (fixed): we write a dhcp-host=MAC,IP line for this MAC. The host gets that exact IP on every renew. Edit the IP and press ✓ to save.
  • Blocked: drops outbound forward traffic from this host. It can still talk to the gateway itself for DHCP/DNS.
  • Tor: marks this host's outbound traffic so it goes through the local Tor TransPort. Requires the Tor service to be running.

Toggle changes affect the database immediately. To translate them into actual nft / dnsmasq rules, press Apply in the topbar.

Danger zone

Reset LAN state. Use this after cloning a template or when the host list got polluted with stale entries: it deletes every row in internal_hosts, empties the dnsmasq lease file (/var/lib/misc/dnsmasq.leases), empties our reservations file, and bounces dnsmasq so clients get fresh leases.

This does not touch peers, peer ACLs, LAN egress rules, users or audit log.
